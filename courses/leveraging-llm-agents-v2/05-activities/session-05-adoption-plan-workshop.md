# Hoạt động 6: 30-Day Skill Adoption Plan

**Buổi:** 5 | **Thời lượng:** 17 phút | **Loại:** Strategic Planning | **Scaffolding:** Independent (Blank template)

---

## Thông tin tổ chức

**Mục tiêu học tập:**
- Xây dựng kế hoạch triển khai skills cho team với ≥3 use case cụ thể theo ngành
- Tổ chức phân công tạo, test, deploy skills

**Hình thức:**
- Breakout rooms theo ngành (HR / Marketing / Finance / Operations)
- 4-5 người mỗi nhóm
- Blank template — nhóm tự fill từ đầu
- Nhóm trình bày 2 phút mỗi cái

**Tài liệu cần chuẩn bị:**
- Blank template (dưới đây)
- Mẫu timeline 30 ngày
- Timer Zoom
- Breakout room assignments

## Hướng dẫn Nhóm

**Breakout rooms (12 phút):**

1. **Chọn use case (2 phút):**
   - Nhóm thảo luận: "3 tác vụ nào ở công việc ngành mình cần automation nhất?"
   - Viết ra, agree on priority

2. **Design skills (3 phút):**
   - Mỗi use case → skill brief table
   - Tên skill, purpose, input/output rõ ràng

3. **Phân công (2 phút):**
   - Ai sẽ creator, tester, deployer, champion?
   - Ghi tên

4. **Timeline 30 ngày (5 phút):**
   - Tuần 1: tạo gì?
   - Tuần 2: test gì?
   - Tuần 3: deploy cách nào?
   - Tuần 4: follow-up + scale?

## Mẫu Timeline (Chi tiết)

### Tuần 1: Build Phase

**Mục tiêu:** Tạo 3 skills base (simple version)

| Ngày | Task | Output | Owner |
|------|------|--------|-------|
| Thứ 2 | Skill 1: YAML + basic body | SKILL.md file | A |
| Thứ 3-4 | Skill 2-3: YAML + basic body | 2× SKILL.md | B, C |
| Thứ 5 | Demo basic: 1 test prompt/skill | Screenshot + feedback | A, B, C |

---

### Tuần 2: Test Phase

**Mục tiêu:** Verify quality + collect feedback

| Task | Test Scenarios | Owner | Success Criteria |
|------|-----------------|-------|------------------|
| Skill 1 test | Scenario A, B, C (từ real work) | D | 3/3 pass, output > 80% quality |
| Skill 2-3 test | Scenario A, B (mỗi skill) | E, F | 2/2 pass, no bugs |
| Document feedback | Ghi down: "tốt ở", "cần sửa" | D, E | Clear list for creator |

---

### Tuần 3: Deploy Phase

**Mục tiêu:** Đưa skill vào sử dụng team

| Task | Activity | Owner | Success Criteria |
|------|----------|-------|------------------|
| Training demo | Show 3 skills trực tiếp (15 min) | G | >80% team join, có Q&A |
| Hands-on practice | Mỗi team member thử 1 skill | Team | Mỗi người chạy ≥1 prompt thành công |
| Setup + access | Enable skills trong workspace | A | All 3 skills activate |

---

### Tuần 4: Scale Phase

**Mục tiêu:** Feedback loop + expansion plan

| Task | Activity | Owner | Output |
|------|----------|-------|--------|
| Collect feedback | Survey / chat: "Skill nào hữu ích? Cần fix gì?" | H | List of 5+ feedback |
| Fix priority issues | Address top 3 bugs/improvements | A, B, C | Updated skills |
| Plan next 3 skills | Identify next use cases (từ feedback) | Team + H | Brief for Tuần 5+ |

## Presentation Format (2 phút/nhóm)

**Mỗi nhóm:**

1. **Intro (15 sec):** "Ngành mình là X. Chọn 3 use case: [list]"
2. **High-impact use case (30 sec):** "Skill #1 này sẽ save [X giờ/tuần] vì [reason]"
3. **Phân công + timeline (45 sec):** "Tuần 1: tạo 3 skills. Tuần 2: test. Tuần 3: deploy. Owner: [names]"
4. **Q&A (15 sec):** "Ai hỏi?"

**Slide deck:** 1 slide = use case table + timeline visual (simple timeline hoặc Gantt)

---

## Debrief & Cross-Team Learning (2 phút)

**GV tổng hợp:**

1. **Pattern:** "Các team chọn skill type nào giống nhau?"
   - Ví dụ: Mỗi team chọn 1 "document generation" skill → có cơ hội share learning

2. **Risk:** "Team nào có risk lớn nhất?" → problem-solve together
   - Ví dụ: "Tester bị busy Tuần 2?" → share tester resource

3. **Timeline:** "Tuần nào bottleneck?" → adjust parallel work

---

## Takeaway

- **Dari cá nhân → team: từ 5 buổi học → 30 ngày deployment.**
- **Áp dụng ngay: không đợi "perfect" — iterative improvement.**
- **Kế tiếp: expand skill library, share best practices, scale org-wide.**
