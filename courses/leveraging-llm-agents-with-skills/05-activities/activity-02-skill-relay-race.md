# Hoạt động 2: "Skill Relay Race"

**Loại hoạt động:** Thực hành có hướng dẫn
**Thời gian:** 20 phút (0:27-0:47 trong buổi 2)
**Chuẩn bị:** Máy tính cá nhân + link Cowork + 3 file mẫu

---

## Mục tiêu

Sau hoạt động này, học viên có thể:
- **Kích hoạt** ít nhất 3 skills khác nhau (pptx, docx, xlsx) trong Cowork
- **Sử dụng** từng skill để hoàn thành 1 tác vụ thực tế trên máy mình
- **Nhận diện** khi nào dùng skill nào bằng cách thực hành
- **So sánh** tính dễ dàng/khó của từng skill thông qua debrief

---

## Thời gian

**Tổng:** 20 phút
- Giới thiệu 3 tác vụ + demo nhanh: 3 phút
- Học viên thực hành (3 task, mỗi task 5 phút): 15 phút
- Debrief + Q&A: 2 phút

---

## Chuẩn bị

**Cho giáo viên:**
1. Upload **3 file mẫu** vào chung shared folder hoặc gửi link download:
   - `task-A-text.txt` (đoạn text ngắn để làm slide)
   - `task-B-meeting-notes.docx` (ghi chú cuộc họp để tóm tắt)
   - `task-C-sales-data.csv` (file CSV bán hàng đơn giản)

2. Chuẩn bị **Cowork instance** đã sẵn sàng (hoặc hướng dẫn học viên log in trước 5 phút)

3. Chuẩn bị **Activity Sheet** (in hoặc link Google Doc) với 3 task

4. Có sẵn **timer** hoặc sử dụng Zoom timer

**Cho học viên:**
- Mở sẵn **Cowork** trên browser
- Tải **3 file mẫu** vào folder cá nhân
- Mở **Activity Sheet** (worksheet dưới đây)

**Tài liệu cần thiết:**
- File mẫu: task-A-text.txt, task-B-meeting-notes.docx, task-C-sales-data.csv
- Activity Sheet (template dưới đây)
- Link hoặc access vào Cowork

## Hướng dẫn từng bước

### Bước 1: Giới thiệu 3 tác vụ (3 phút)

**Giáo viên nói:**

> "Bạn sẽ thực hành 3 tác vụ **khác nhau**, mỗi cái dùng **1 skill khác nhau**. Không cần phải hoàn hảo — mục tiêu là bạn **kích hoạt skill**, **gõ prompt**, **nhận output**, và **thấy kết quả**.
>
> Tất cả bạn làm **trên máy mình**. Tôi sẽ support qua chat nếu bạn bị stuck. Bạn có **5 phút** cho mỗi task.
>
> **Task A — Tạo slide từ text (5 phút)** — Dùng skill **pptx**
> Task B — Tạo handout từ ghi chú (5 phút) — Dùng skill **docx**
> Task C — Phân tích bản bán hàng (5 phút) — Dùng skill **xlsx**"

**Sau đó, giáo viên demo nhanh 1-2 bước của Task A (1 phút):**

**Demo Task A (Live):**

Giáo viên share màn hình:
1. Mở Cowork
2. Kéo file `task-A-text.txt` vào chat window

### Bước 2: Học viên thực hành 3 Task (15 phút)

Học viên làm **tuần tự** trên máy mình. Mỗi task 5 phút. Giáo viên monitor qua chat/breakout.

#### **TASK A: Tạo slide từ text — Dùng skill PPTX (5 phút)**

**Chuẩn bị:**
- Tải file `task-A-text.txt` (nội dung mẫu dưới đây)

**Nội dung file `task-A-text.txt` (mẫu):**
```
Bản tin: Xu hướng tuyển dụng 2026

Năm 2026, thị trường tuyển dụng sẽ có 5 xu hướng chính:

1. Remote work là mặc định: 70% công ty sẽ chấp nhận làm việc từ xa ít nhất 2 ngày/tuần.

2. AI skills bắt buộc: Mỗi vị trí công việc sẽ yêu cầu ít nhất 1 kỹ năng liên quan AI (prompt writing, data analysis, automation).

3. Reskilling programs: Công ty đầu tư vào đào tạo lại kỹ năng nhân viên cũ thay vì tuyển dụng người mới.

4. Diversity là tiêu chí hàng đầu: 80% công ty tìm kiếm ứng viên từ các nhóm đa dạng.

5. Flexible benefits: Lương cao đã không đủ — phúc lợi linh hoạt (chọn bảo hiểm, thời gian, địa điểm) là yếu tố quyết định.
```

**Hướng dẫn học viên:**

1. **Mở Cowork** trong browser
2. **Kéo file `task-A-text.txt`** vào chat area (hoặc copy-paste nội dung)
3. **Gõ prompt:**
   ```
   Hãy sử dụng skill pptx để tạo 1 slide deck từ dữ liệu này.

   Yêu cầu:
   - Slide 1: Title slide — "Xu hướng tuyển dụng 2026"
   - Slide 2: 5 key points (mỗi dòng 1 bullet)
   - Slide 3: Call to action — "Chuẩn bị kỹ năng AI hôm nay"

   Design: Màu sắc chuyên nghiệp, readable, có icon/emoji nhấn mạnh.
   ```

4. **Chạy prompt** (click play/enter)

5. **Chờ kết quả** — Cowork sẽ tạo file .pptx

6. **Download file** xuống máy tính

7. **Mở file** trong PowerPoint/Google Slides để check kết quả

8. **Check worksheet:** Đánh dấu ✓ "Đã hoàn thành" task A

**Nếu gặp khó khăn:**
- Lỗi "Skill pptx không tìm thấy?" → Kiểm tra skill đã được kích hoạt chưa
- Kết quả không đẹp? → Thử lại prompt, chi tiết hơn
- Timeout? → Chờ 1-2 phút hoặc liên hệ giáo viên

#### **TASK B: Tạo handout từ ghi chú — Dùng skill DOCX (5 phút)**

**Chuẩn bị:**
- Tải file `task-B-meeting-notes.docx` (nội dung mẫu dưới đây)

**Nội dung file `task-B-meeting-notes.docx` (mẫu):**
```
CUỘC HỌP: Kế hoạch Q2 2026 — Marketing Team
Ngày: 11/3/2026 | Người dự: Hana (Lead), Tom, Lisa, Paul

GHI CHÚ NHANH:
- Q2 focus: TikTok + Instagram. Giảm Facebook.
- Budget: Tăng từ 500k lên 700k (thêm 200k cho content creator)
- Target: 500k followers by end Q2 (hiện 300k)
- Content pillars: 60% product demo, 30% lifestyle, 10% behind-the-scenes
- Hiring: Tuyển 2 junior content creator + 1 community manager by April
- Risk: TikTok algorithm thay đổi. Solution: Đa dạng format (Shorts, Reels, Stories)
- Deadline review: 15/4 (mid-quarter check-in)

ACTION ITEMS:
- Hana: Present plan to CEO by 15/3
- Tom: Prepare content calendar for April-June
- Lisa: Source 3 content creator agencies for hiring
- Paul: Set up analytics dashboard for tracking followers
```

**Hướng dẫn học viên:**

1. **Mở Cowork**
2. **Kéo file `task-B-meeting-notes.docx`** vào chat (hoặc copy nội dung)
3. **Gõ prompt:**
   ```
   Hãy sử dụng skill docx để tạo 1 handout chuyên nghiệp từ ghi chú cuộc họp này.

   Yêu cầu:
   - Tiêu đề + ngày tháng
   - 3 section: Tóm tắt (summary), Action Items (danh sách), Risks & Solutions
   - Format: Professional, dễ in, có đủ margin
   - Heading rõ ràng, bullet points

   Tên file: Marketing-Q2-Handout.docx
   ```

4. **Chạy prompt**

5. **Download file**

6. **Mở file** trong Word/Google Docs để check

7. **Check worksheet:** Đánh dấu ✓ task B

**Nếu gặp khó khăn:**
- Kết quả format lạ? → Prompt yêu cầu "professional" hơn
- File không tạo được? → Thử prompt đơn giản hơn, không quá chi tiết

### Bước 3: Debrief (2 phút)

**Giáo viên hỏi (pick 3 người):**

1. **"Công việc nào dễ nhất? Tại sao?"**
   - Dự kiến trả lời: Thường là Task A (pptx) vì prompt đơn giản

2. **"Công việc nào khó nhất? Khó như thế nào?"**
   - Dự kiến: Task C (xlsx) vì data analysis cần prompt cụ thể hơn

3. **"Có cái gì bất ngờ bạn không?"**
   - Dự kiến: Bất ngờ output chất lượng cao / Bất ngờ skill có giới hạn

**Giáo viên tóm tắt:**

> "Bạn vừa thấy 3 skills khác nhau. Mỗi skill chuyên môn ở 1 loại công việc. pptx → slide, docx → text, xlsx → data.
>
> Điều quan trọng: bạn không phải là 'lập trình viên' để dùng skills. Bạn chỉ cần **kích hoạt skill + gõ prompt rõ ràng = output tốt**.
>
> Buổi 3, chúng ta sẽ viết **skill riêng của bạn** cho công việc của chính bạn."

## Debrief / Kết nối nội dung

**Kết nối nội dung từ hoạt động:**

> "Tất cả bạn vừa dùng skills mà **người khác tạo** từ trước. Và bạn thấy: nó HOẠT ĐỘNG. Bạn không cần là developer, bạn chỉ cần gõ prompt.
>
> Bây giờ, câu hỏi là: **'Khi skills có sẵn không đủ cho công việc RIÊNG của tôi thì sao?'**
>
> Ví dụ: skill docx tạo handout chung chung. Nhưng công ty bạn có template báo cáo rất cụ thể — 5 section, font cố định, logo, footer, v.v. **Bạn cần skill riêng.**
>
> **Buổi 3 sắp tới, bạn sẽ VIẾT CHÍNH cái skill đó.** Không khó như bạn tưởng. Bạn sẽ dùng template, điền vào, test, done. Ready?"

## Lưu ý dành cho giáo viên

1. **File mẫu:** Tạo 3 file đơn giản (không quá phức tạp) để tránh overwhelm

2. **Support mode:** Có chuẩn bị câu trả lời cho các lỗi thường gặp:
   - "Skill không tìm thấy?" → Check xem skill đã enabled chưa
   - "Timeout?" → Chờ thêm hoặc thử prompt ngắn hơn
   - "Output không tốt?" → Thử prompt chi tiết hơn

3. **Timing:** Nếu ai hoàn thành sớm hơn 5 phút, hỏi: "Bạn sẽ dùng skills này cho công việc gì trong tuần?"

4. **Inclusive:** Ai tech-phobic có thể hỏi giáo viên / peer help tạm thời. Đây không phải test, là practice.

5. **Save outputs:** Gợi ý học viên lưu lại 3 file output — họ sẽ cần xem lại khi review buổi 2.

## Success Criteria

Hoạt động thành công khi:
- ✓ Tối thiểu **70%** học viên hoàn thành cả 3 tasks
- ✓ Mỗi task output có kết quả **tangible** (file đúng định dạng, có nội dung)
- ✓ Học viên có thể giải thích **"Khi nào dùng skill nào"** (pptx cho slide, docx cho text, xlsx cho data)
- ✓ Feedback debrief cho thấy: skill nào dễ, skill nào khó, và **lý do tại sao**
