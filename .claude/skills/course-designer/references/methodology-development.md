# Methodology: Development — Slides and Activities

## 5.1 Slide Design Principles

**Principle 1 — Easy to Read (KISS)**: Keep It Short and Simple. Minimal text.

**Principle 2 — Easy to Understand**: Use meaningful images to replace text. Visuals aid comprehension and memory.

**Principle 3 — Easy to Remember**: Highlight key content using large fonts, bold/contrasting colors.

**Principle 4 — Dynamic/Engaging**:
- Combine warm and cool colors for contrast
- Warm colors ≈ 30% of slide (for focal points)
- Consistent color usage (same color for same content type)
- Use warm colors to draw attention to key elements

### Slide Specifications (Concrete Numbers)

These translate design principles into measurable specs:

- **Slides per session:** 8-12 content slides per 60-min session (excluding title/closing). 1 slide per 5-7 min of instruction.
- **Text density:** Slide shows the KEY POINT only. If you can remove text and the facilitator can still teach from it, the slide is good. If removing text makes it unusable, too much text.
- **Font sizes (virtual delivery):** Minimum 24pt body text, 32pt headers. Smaller is unreadable on shared screens.
- **Color ratio:** 70% cool tones (background, secondary info), 30% warm (focal points, CTAs, key icons). Every CTA/action slide uses warm accent.
- **Animation:** Only for progressive disclosure (reveal bullet by bullet). Never for decoration.
- **Accessibility:** Alt-text guidance for every image. No meaning by color alone. WCAG AA contrast (4.5:1 text, 3:1 large text).

**Slide Production Checklist**:
- Create Slide Master for consistency
- Select illustrative images for content
- Link media (video, audio) where helpful
- Use SmartArt for diagrams/processes
- Add purposeful animations (progressive disclosure only)
- One key message per slide
- Verify warm/cool color ratio (~30% warm)
- Ensure WCAG AA color contrast (4.5:1 for text, 3:1 for large text)
- Never convey meaning through color alone
- Test readability at shared-screen resolution

## 5.2 Learning Activity Design

Seven main activity types:

**1. Group Discussion**: Share experiences, explore perspectives. Pose question → small group → plenary → synthesis.

**2. Case Study**: Apply knowledge to realistic scenarios. Present case → analysis → solutions → debrief.

**3. Practice/Exercise**: Hands-on skill application. Demo → guided practice → independent practice → feedback.

**4. Role Play**: Practice interpersonal skills safely. Set scenario → assign roles → perform → observe → debrief.

**5. Crossword/Word Games**: Reinforce terminology. Present puzzle → individual/pair work → answers → discuss.

**6. Brainstorming**: Generate ideas. State topic → free generation (no criticism) → organize → prioritize.

**7. Mind Mapping**: Organize concepts visually. Central concept → branches → sub-branches → cross-connections → present.

### Activity Design Criteria (REAL)
Every activity must be:
- **R**elevant: Directly relates to course content and learner context
- **E**ngaging: Captures interest and motivates participation
- **A**ctive: Learners are doing, creating, solving — not just watching
- **L**earner-centered: Builds on and leverages learner experience

### Facilitator Micro-Actions (Required for Every Activity)

Activities often have 10-20 minutes of "learner working time" where the procedure just says "learners work in groups." But what does the facilitator DO during this time? Without explicit guidance, facilitators either hover awkwardly or disengage entirely — both hurt the learning experience.

Every activity procedure must include a **"During Activity"** section:

- **Roaming schedule:** Which breakout rooms/groups to visit, in what order, for how long. Example: "Visit each room for 2 min. Start with quietest room."
- **What to listen for:** Common misconceptions, off-track signals, moments to praise. Example: "If learners confuse 'agent' with 'chatbot', redirect using the GPS analogy."
- **Intervention triggers:** When to step in vs. let learners struggle productively. Example: "If group hasn't started after 2 min, prompt: 'Start with the pre-filled example — what pattern do you see?'"
- **Early finisher instructions:** What to do if a group finishes ahead of time. Example: "If done early, try the extension challenge: 'Can you find a task that SHOULDN'T be given to an agent?'"
- **Time-check announcements:** Specific cues at specific times. Example: "At 5 min remaining, announce: 'Start wrapping up — pick your best example for sharing.'"

These micro-actions are what separate a facilitator guide from a worksheet.

## 5.3 Facilitator Guide: Equal Depth Rule

The facilitator guide is the instructor's primary tool during delivery. If Session 1 gets a full timing table, speaker scripts, and troubleshooting scenarios but Sessions 2-5 get one-line summaries, the guide fails its purpose.

**Equal Depth Rule:** Every session section in the facilitator guide MUST contain:
1. **Timing table** — minute-by-minute breakdown matching the lesson plan
2. **Speaker script suggestions** — opening lines + key transition phrases (not a full script, but enough to start)
3. **Technical setup notes** — what to have open, what to share, backup plan
4. **Troubleshooting scenarios** — at least 2 per session (what if X goes wrong?)
5. **Engagement checkpoints** — where to read the room, how to adjust pace

**Length balance:** If a guide covers N sessions, each session section should be roughly equal in length (±20%). A 10:1 ratio between any two sessions means the shorter one needs expansion.

**Why this matters:** Facilitators don't just teach Session 1 — they teach all sessions. And Sessions 3-5 (where learners are more independent) actually need MORE facilitation nuance, not less, because the facilitator's role shifts from "presenter" to "coach."

## 6. Implementation

Pilot with small group before full rollout. Observe what works, where timing is off. Collect feedback on content relevance, activity effectiveness, overall experience.

## 7. Evaluation and Revision

Review holistically after implementation. Use the 19-criterion checklist (`evaluation-criteria-part-1.md` + `evaluation-criteria-part-2.md`). Revise based on findings, re-pilot if changes are significant.
