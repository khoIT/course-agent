# Course Design Guideline

A concise, theory-grounded reference for designing effective professional learning experiences. Consolidates instructional design foundations, adult learning theory, evidence-based teaching strategies, and evaluation frameworks into actionable principles.

Load this reference when you need the *theory* behind design decisions. For phase-specific *procedures*, see the other methodology references.

---

## 1. Foundational Theories

### 1.1 Andragogy — How Adults Learn (Knowles)

Adults learn fundamentally differently from children. Malcolm Knowles identified 5 assumptions:

| Assumption | Implication for Design |
|------------|----------------------|
| **Self-directed** | Give learners choices, not just instructions. Let them set pace within structure. |
| **Experience-rich** | Build on what they already know. Use their work as learning material. |
| **Ready when relevant** | Connect every concept to an immediate work problem. "You'll use this Monday." |
| **Problem-oriented** | Organize by tasks to solve, not topics to cover. |
| **Internally motivated** | Show personal benefit (WIIFM) early. Avoid grades — use real-world success criteria. |

**Design rule:** If an activity doesn't connect to something the learner does at work, cut it.

### 1.2 Bloom's Taxonomy — Depth of Learning

Six cognitive levels, from shallow to deep. Most corporate training targets levels 2-4.

| Level | Verb Examples | Course Design Use |
|-------|--------------|-------------------|
| 1. **Remember** | List, define, recall | Pre-reading, glossaries — don't spend session time here |
| 2. **Understand** | Explain, compare, classify | Quick concept intros (3-5 min max before practice) |
| 3. **Apply** | Demonstrate, use, implement | Hands-on activities — where 60%+ of time should go |
| 4. **Analyze** | Compare, diagnose, distinguish | Case studies, troubleshooting exercises |
| 5. **Evaluate** | Judge, critique, recommend | Peer review, quality assessment activities |
| 6. **Create** | Design, build, produce | Capstone projects, portfolio activities |

**Writing objectives:** Always use an action verb from the target level + standard + condition.
- Bad: "Understand customer service principles" (level 2 verb, no standard)
- Good: "Apply the HEARD framework to de-escalate 3 complaint types within a 5-min interaction" (level 3, measurable)

### 1.3 Merrill's First Principles of Instruction

Five evidence-based principles for effective instruction. Use as a design checklist:

1. **Task-centered** — Organize around real-world tasks, not abstract topics. "Create a contract" not "Learn about contracts."
2. **Activation** — Activate prior knowledge before introducing new content. "What do you already know about X?"
3. **Demonstration** — Show before asking learners to do. Demo → guided practice → independent practice.
4. **Application** — Learners must practice with feedback. Watching demos isn't learning.
5. **Integration** — Learners reflect and transfer to their real context. "How will you use this Monday?"

**Design rule:** Every session should hit all 5 principles. If you skip Application, you have a lecture, not a course.

## 2. Evidence-Based Teaching Strategies

### 2.1 Four Strategies with Strong Evidence (1,500 papers meta-analysis)

| Strategy | What It Is | Effect Size | How to Apply |
|----------|-----------|-------------|--------------|
| **Clear instructions** | Explicit, specific task descriptions | High | Brief with role + task + format + constraints |
| **Structured output** | Templates, formats, frameworks | High | Provide templates learners fill in, not blank pages |
| **Few-shot examples** | 2-3 worked examples before practice | High | Show good + bad examples side-by-side |
| **Chain-of-thought** | Step-by-step reasoning walkthrough | High | Model expert thinking aloud, then ask learners to do same |

### 2.2 Retrieval Practice (Dr. Agarwal)

Pulling information OUT of memory is more effective than re-reading. Apply:
- **Session starts:** "What do you remember from last session?" before any review
- **During sessions:** Quick recall quizzes (not graded — purely for consolidation)
- **Between sessions:** Spaced challenges at Day 3, 7, 14, 30

### 2.3 Spacing and Interleaving

- **Spacing:** Distribute practice over time. A single 8-hour day is less effective than four 2-hour sessions.
- **Interleaving:** Mix problem types within practice sessions. Don't do all "type A" then all "type B" — interleave them.

### 2.4 The 60% Rule

At least 60% of session time should be hands-on practice. Maximum 15 minutes of lecture before an activity. If you're talking for 20 minutes straight, you've lost them.

## 3. Activity Design Principles

### 3.1 REAL Criteria

Every activity must be:
- **R**elevant — connected to learner's actual work
- **E**ngaging — sustains attention, not busywork
- **A**ctive — learners DO something, not watch
- **L**earner-centered — designed for their level, not the facilitator's comfort

### 3.2 Tiered Activities (Mixed Skill Levels)

Same scenario, three depths:
- **Foundation:** Follow the provided template step-by-step
- **Extension:** Adapt the template to your own use case
- **Challenge:** Create from scratch using advanced techniques

This solves the "too easy for some, too hard for others" problem without separate tracks.

### 3.3 Bad-to-Good Comparative Method (UVic)

Learners produce a weak version first, then refine with new techniques, then compare side-by-side. The before/after contrast makes improvement tangible and memorable. Works especially well for skills where "quality" is subjective (writing, design, communication).

### 3.4 Facilitator Micro-Actions

Every activity needs a "During Activity" section — what the facilitator does while learners work:
- **Roaming schedule:** Check in with 3-4 learners per activity
- **Intervention triggers:** Signs a learner is stuck (no progress after 3 min, repeated same error)
- **Early-finisher tasks:** Challenge extension for fast learners
- **Time checks:** "5 minutes left" announcements

Without micro-actions, facilitators stand awkwardly at the front during practice time.

## 4. Content Architecture

### 4.1 WHY-WHAT-HOW-SO WHAT Sequencing

For each concept or module:
1. **WHY** — Why does this matter? Connect to learner's work problem.
2. **WHAT** — What is the concept/technique? Brief explanation (3-5 min).
3. **HOW** — How do you apply it? Demo + guided practice.
4. **SO WHAT** — What changes in your work now? Reflection + transfer planning.

### 4.2 Content Selection: Exclude, Don't Include

The hardest part of course design is deciding what to leave out. Learners retain 3-5 key concepts per session, not 15.

**MoSCoW method:**
- **Must have:** Core skills they can't leave without
- **Should have:** Important but can be handout-only
- **Could have:** Nice context — cut if time-pressured
- **Won't have:** Interesting but off-scope — explicitly exclude

**Rule:** If you can't explain why a piece of content is "Must Have" in one sentence, it's not.

### 4.3 The Throughline Example

Use a single, realistic scenario that runs across the entire course. Each session builds on it.
- Session 1: Simple version with basic tools
- Session 2: Same scenario, more sophisticated approach
- Session 3: Hands-on execution of the scenario
- Session N: Learners adapt the pattern to their own work

This provides continuity, reduces cognitive load (no new context each session), and shows progression.

## 5. Evaluation Framework

### 5.1 Kirkpatrick's Four Levels

| Level | Measures | How | When |
|-------|----------|-----|------|
| **1. Reaction** | Did they like it? Was it relevant? | Post-session survey, NPS | Immediately after |
| **2. Learning** | Did they learn the skills? | Pre/post comparison, rubric-scored output | During + after |
| **3. Behavior** | Are they using it at work? | 30-day follow-up survey, manager feedback | 2-4 weeks after |
| **4. Results** | Did it impact the business? | Productivity metrics, time saved | 1-3 months after |

**Design implication:** Plan for Level 3 at design time, not after delivery. Build in spaced practice, buddy systems, and follow-up touchpoints.

### 5.2 Post-Training Transfer Design

Training that ends at the classroom door fails. Design for transfer:
- **Spaced Practice:** Follow-up challenges at Day 3, 7, 14, 30
- **Prompt Library / Job Aids:** Reusable references organized by task
- **Buddy System:** Pair learners for 2-week check-ins
- **Manager Briefing:** 1-page summary for managers on what learners can now do

## 6. Online Delivery (Zoom/Virtual)

### 6.1 Engagement Techniques

| Technique | How It Works | Frequency |
|-----------|-------------|-----------|
| **Polls** | Quick multiple-choice to check understanding | Every 8-10 min |
| **Chat waterfall** | Everyone types answer, sends simultaneously on "Go" | 2-3 per session |
| **Screen share show-and-tell** | Learner shows work, class suggests improvements | Block C |
| **Two-way dialogue** | Read participant responses aloud, respond | Throughout |

### 6.2 Energy Management

- **90-minute maximum** without break. 10-min break every 45-50 min.
- **Vary modality every 15 min:** lecture → demo → practice → share → discussion
- **Camera-off breaks:** Explicitly allow cameras off during breaks
- **Movement prompts:** "Stand and stretch while AI processes"

### 6.3 Session Pacing Template

```
0:00-0:35  Block A — Concept + Demo (max 5 min lecture, then practice)
0:35-0:40  Break 1
0:40-1:15  Block B — Hands-on Practice (tiered: Foundation/Extension/Challenge)
1:15-1:20  Break 2
1:20-1:50  Block C — Apply to Own Work + Showcase + Q&A
1:50-2:00  Block D — Homework Preview + Wrap
```

## 7. Quality Signals (Quick Self-Check)

Before finalizing any course material, verify:

- [ ] Every design decision traces to the learner portrait
- [ ] Learning objectives use action verbs with standards and conditions
- [ ] Content is organized by task/problem, not by topic
- [ ] 60%+ of session time is hands-on practice
- [ ] Activities are tiered for mixed skill levels
- [ ] Facilitator guide has equal depth across ALL sessions
- [ ] Learner handout is standalone-usable (organized by task, not session)
- [ ] Post-training transfer is designed in (spaced practice, job aids, buddy system)
- [ ] Claims are fact-checked, not assumed
- [ ] A throughline example connects all sessions
