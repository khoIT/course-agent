# Methodology: Philosophy & Analysis

> Based on ADDIE instructional design model and learner-centered training principles.

## 1. Learner-Centered Training Philosophy

The learner-centered approach focuses on what learners need to learn, not what the trainer wants to teach.

### Four Core Principles

1. **Focus on content learners NEED**: Identify problems learners need to solve. Provide only essential content.

2. **Deliver content matching learner styles**: Visual, Auditory, Kinesthetic. Use diverse delivery methods.

3. **Create opportunities for participation and discovery**: Learners retain more when actively participating. (Edgar Dale: 10% read, 20% hear, 30% see, 50% hear+see, 70% say, 90% say+do)

4. **Highlight benefits (WIIFM — What's In It For Me)**: State course benefits early to motivate participation.

### Interactive Learning
Maximize multi-directional interaction: Trainer↔Learner, Learner↔Learner. More interaction dimensions = better outcomes.

## 2. ADDIE Model Overview

Five phases: **Analysis → Design → Development → Implementation → Evaluation & Revision**

Each phase produces outputs feeding the next:
- Analysis → Learner profile
- Design → Objectives + Content outline + Lesson plan
- Development → Slides + Learning activities + Materials
- Implementation → Pilot delivery
- Evaluation → Improvement recommendations

## 3. Step 1: Analysis — Learner Profiling

### Why Analyze Learners?
Helps select appropriate objectives/content, diversify activities for learning styles, leverage learner experience.

### Learner Groups
- **Primary Learners**: Main target group
- **Secondary Learners**: Others who may benefit; don't compromise primary group's needs

### What to Analyze

**Training Needs**: Fill knowledge/skill gaps, solve workplace problems, acquire skills for new responsibilities.

**Learning Styles**:
- **Visual**: Reading, images, videos, diagrams
- **Auditory**: Lectures, discussions, shared experiences
- **Kinesthetic**: Games, group work, case studies, hands-on practice

**Learning Conditions**: Time available, environment (classroom/online/on-the-job)

### Learner Portrait (KSA Profile)
Three dimensions:
- **Attitude (A)**: Perspectives and feelings about the training topic
- **Skills (S)**: Current abilities and competencies
- **Knowledge (K)**: Existing understanding of subject matter

The portrait informs ALL subsequent design decisions: content inclusion/exclusion, complexity, methods, practice amount.

### Example Learner Portrait

```
## Learner Portrait: Customer Service Reps — Conflict De-escalation

### Target Learners
- Primary: Frontline reps (6-18 months), retail banking
- Secondary: Team leads who coach reps

### Current State (KSA)
- Knowledge: Company policies + basic scripts. No formal conflict resolution training.
- Skills: Handle routine complaints. Struggle with emotional callers — escalate to supervisor.
- Attitudes: Motivated but anxious. Past negative experiences eroded confidence.

### Learning Context
- Styles: Predominantly kinesthetic — role-play essential
- Time: 1 full day (8 hours)
- Environment: In-person, groups of 12-16
- Motivation: High — want tools for tough calls. WIIFM is clear.

### Design Implications
- Use real call examples (anonymized) over abstract theory
- ≥40% time on activities (role-play, practice)
- Cognitive level: Apply (Level 2)
- Include desk "cheat sheet" handout
```
