# Methodology: Design — Objectives, Content, Lesson Plan

## 4.1 Writing Learning Objectives

Objectives must be learner-centered: what LEARNERS will be able to do.

**Benefits**: Learners know expectations, trainer stays focused, content has selection criteria, evaluators can measure results.

**Three Cognitive Levels** (simplified Bloom's):

| Level | Description | Suitable For |
|-------|-------------|-------------|
| 1. Understand & Retain | Grasp theory without applying | Support roles |
| 2. Apply | Perform tasks or complete jobs | Execution roles |
| 3. Problem-Solve | Analyze, evaluate, propose improvements | Management roles |

**Formula**: Action Verb + Standard/Criteria + Condition

- **Action Verb**: Observable, measurable. AVOID: "understand", "know", "appreciate". USE: "list", "demonstrate", "create", "analyze"
- **Condition**: Context (e.g., "without supervisor support", "using company template")
- **Standard**: Performance level (e.g., "99% accuracy", "following company guidelines")

**Verbs by Level** (top 10 each):
- Level 1: List, Describe, Explain, Identify, Compare, Classify, Summarize, Define, Distinguish, Illustrate
- Level 2: Demonstrate, Implement, Use, Prepare, Select, Modify, Apply, Draft, Complete, Assemble
- Level 3: Evaluate, Analyze, Design, Propose, Organize, Plan, Produce, Distinguish, Prioritize, Justify

### Example
**Bad:** "Learners will understand conflict de-escalation techniques" (not observable)
**Good:** "Demonstrate at least 3 de-escalation techniques during a simulated angry-customer call, following the CALM framework, with no policy violations"

## 4.2 Selecting and Categorizing Content

**Three Categories** (concentric circles):
- **Must Have** (core): Without this, learners CANNOT achieve objectives. Always include.
- **Should Have** (middle): Supports easier achievement. Include if time permits.
- **Nice to Have** (outer): Useful but doesn't significantly impact objectives. Include only with ample time.

## 4.3 Content Sequencing

**WHY-WHAT-HOW-SO WHAT Framework**:
1. **WHY**: Why important? What problem does it solve?
2. **WHAT**: Present the content clearly
3. **HOW**: Practice — guide and create application opportunities
4. **SO WHAT**: Real-world application reminders and caveats

**Delivery Techniques**:
- **Inductive (bottom-up)**: Examples → general principles. Good for experienced learners.
- **Deductive (top-down)**: Principles → specific cases. Good for new content.

## 4.4 Content Depth Calibration

How deep to go depends on WHO is learning. Use these heuristics when building the lesson plan:

### Minutes-per-Concept by Learner Type

| Learner Type | Min/Concept | Rationale |
|---|---|---|
| Non-technical (support, ops, sales) | 3–4 min | Need more context, analogies, real-world grounding |
| Technical (developers, engineers) | 2 min | Absorb abstractions faster, want to get to hands-on |
| Mixed groups | 3 min | Split the difference; use layered explanations |
| Leadership/executive | 2–3 min | Want strategic framing, skip implementation details |

### Guardrails

- **Max 4 new concepts per 15-min segment.** Beyond this, retention drops sharply — learners start confusing concepts with each other.
- **Virtual delivery: interaction every 8 minutes.** After 8 min of lecture, insert a poll, chat prompt, or pair share. No exceptions for virtual sessions.
- **Every explanation needs 3 anchors:** 1 analogy (maps to something familiar), 1 visual (diagram, screenshot, or demo), 1 industry example (real company, real scenario). If you can't find all three, the concept may be too abstract for the audience.
- **The "explain to a colleague" test:** If a learner couldn't explain this concept to a colleague over coffee after the session, the content is too dense or too shallow. Calibrate accordingly.

### Applying Depth Calibration to Lesson Plans

When building timing tables in the lesson plan:
1. Count the new concepts per section
2. Multiply by the min/concept for your learner type
3. Add activity time (typically 1.5× the concept time for practice)
4. If total exceeds session time, CUT concepts (move to "Should Have" or "Nice to Have") — never compress time

**Example:** 5 new concepts for non-technical learners = 5 × 4 min = 20 min instruction + 30 min practice = 50 min. Fits a 60-min session with 10 min for intro/wrap.

## 4.5 Building the Lesson Plan

The facilitator's roadmap. Must specify:
- Session flow from start to finish
- Key content points + delivery method (inductive/deductive)
- Important emphasis points during explanation
- Discussion questions for participation
- Activities for practicing/applying content
- Estimated time per section
- Supporting resources (slides, handouts, references)

**Balance**: Not too brief (facilitators get lost) nor too detailed (kills creativity).
