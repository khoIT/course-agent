# Scaffolding Techniques for Non-Technical Adult Learners in AI/LLM Training

**Date:** March 12, 2026
**Course:** Leveraging LLM Agents (Vietnamese, 5×1hr)
**Target:** Non-technical business users

---

## Key Scaffolding Techniques

### 1. Micro-Stepping & Chunking
- **Break instruction into 5-10 minute segments** with activity or check between each segment
- Short lessons enable brain connection between prior knowledge and new information
- Cognitive load research shows adults process better with focused, manageable chunks
- Within each chunk: model skill → guided practice → independent attempt

**Source:** [U.S. Department of Education LINCS - Designing Technology for Adult Learners](https://lincs.ed.gov/professional-development/resource-collections/profile-1021)

### 2. "I Do, We Do, You Do" Progression
- **I Do:** Instructor models the task (demonstrate LLM agent workflow step-by-step)
- **We Do:** Guided practice together (fill-in-the-blank exercises, instructor explains reasoning)
- **You Do:** Semi-guided independent work (templates provided, learner completes with support available)

This progression builds confidence before full autonomy. Adults develop self-discipline, problem-solving, and decision-making through ownership.

**Source:** [TeachHUB - Guided vs. Independent Practice](https://www.teachhub.com/professional-development/2020/08/guided-vs-independent-practice/)

### 3. Guided Templates & Scaffolded Worksheets
- Provide **fill-in-the-blank prompt templates** for common agent tasks
- Pre-structured worksheets with labeled sections: [Input] → [Agent Process] → [Expected Output]
- Gradually remove scaffolds: Session 1 = mostly filled, Session 5 = learner creates from scratch
- Adult learners connect new skills to job-relevant tasks when templates reflect real workflows

**Source:** [Education Northwest - Effective Design for Adult Learners](https://educationnorthwest.org/insights/effective-design-adult-learners)

### 4. Checkpoints in 60-Minute Sessions
Optimize engagement within 1-hour virtual windows:

| Time | Activity | Purpose |
|------|----------|---------|
| 0-5 min | Anchor to prior knowledge | Connect to previous lesson/job role |
| 5-15 min | Modeled instruction (I Do) | Show agent workflow live |
| 15-25 min | Guided practice + chat poll | We Do + quick comprehension check |
| 25-35 min | Semi-guided activity in breakout rooms | You Do with peer support |
| 35-45 min | Full-group debrief + Q&A | Surface blockers, normalize struggles |
| 45-60 min | Independent challenge + reflection | Self-assess, commit to next action |

**Critical:** After 8-10 minutes of passive content, insert poll, chat question, or breakout activity.

**Source:** [University of Rochester - Learning Activities for Synchronous Online Classes](https://www.rochester.edu/college/teaching/teaching-online/synchronous-learning-activities.html), [BC CTE - Planning an Online Synchronous Session](https://cteresources.bc.edu/documentation/synchronous-teaching-considerations/planning-an-online-synchronous-session/)

### 5. Checks for Understanding
- **Polls:** "Which step comes next?" (multiple choice)
- **Chat prompts:** "Type one word: ready or confused" (1-2 sec feedback)
- **Breakout debrief:** "Show me your result" (peer-to-peer validation)
- **Reflection:** "What will you try first in your role?" (job transfer check)

Adult learners benefit from low-stakes, immediate feedback without judgment.

**Source:** [Montclair State ITDS - Synchronous Teaching Strategies](https://www.montclair.edu/itds/digital-pedagogy/online-and-hybrid-teaching/synchronous-teaching-strategies/synchronous-class-session-examples-and-strategies/)

## LLM-Specific Scaffolding Insights

### Prompt Engineering as Scaffolded Skill
- Begin with **pre-written, annotated prompts** (identify each element: role, task, constraint, output format)
- Guided practice: Learners modify ONE variable in template (e.g., change task description, keep role fixed)
- Semi-guided: Provide role + output format; learner writes task description
- Independent: Learner builds prompt from blank template with checklist guide

This mirrors the ephemeral UI scaffold pattern where intermediate steps (templates) reduce prompt engineering complexity.

**Source:** [Apple ML Research - BISCUIT: Scaffolding LLM-Generated Code](https://machinelearning.apple.com/research/biscuit-scaffolding-llm)

### Agent Task Scaffolding (Cognitive Framework)
Use Evidence-Centered Design: Break complex agent tasks into cognitive micro-steps:
- Step 1: Identify the problem (What are you trying to solve?)
- Step 2: Select the agent (Which pre-built skill does this?)
- Step 3: Frame the request (What context does the agent need?)
- Step 4: Evaluate output (Is the result useful? Why/why not?)
- Step 5: Iterate (What would you ask differently?)

**Source:** [Springer - A Theory of Adaptive Scaffolding for LLM-Based Pedagogical Agents](https://arxiv.org/html/2508.01503v1)

## Practical Activity Progressions for 5×1hr Course

### Session 1-2: Foundations (Fully Guided)
- Model: Show a live agent workflow end-to-end
- Guided: Learners follow along with annotated worksheet
- Checkpoint: Poll "What is the agent doing here?" (3 options)
- Activity: Fill-in-the-blank: "To get [OUTCOME], I would tell the agent [BLANK]"

### Session 3: Emerging Independence (Semi-Guided)
- Model: Brief demo of new agent skill
- Guided: Pair activity in breakout rooms (one reads prompt, one types response, swap roles)
- Checkpoint: Chat: "What's one thing you'd change in that prompt?"
- Activity: Template + 2 blank fields to complete (role stays fixed)

### Session 4-5: Applied Independence (Minimal Scaffolding)
- Model: 3-minute demo of advanced use case
- Guided: Peer feedback on learner-created prompts (structured feedback template provided)
- Checkpoint: "Share your result in chat" (peer validation)
- Activity: Blank template; learner builds full prompt; self-checklist for evaluation

## Vietnamese Course Adaptation Notes

- **Respect for experience:** Open each session asking "What's a task in your role you'd love to automate?" before showing agent examples
- **Relevance first:** Use business/operational scenarios (expense tracking, meeting notes, proposal review) not generic examples
- **Low-tech breakout rooms:** Pair learners for peer support; assign one as "guide" (native agent experience) and one as "navigator" (new user)
- **Asynchronous fallback:** Record 3-minute clips of each "I Do" segment; learners can rewatch before breakout activities

---

## Summary: Recommended Session Design
**Per 60-min session:** 10min model → 10min guided practice → 10min checkpoint + debrief → 20min semi-guided activity → 10min reflection
**Progression:** Full template (S1) → Partial template (S3) → Blank template + checklist (S5)
**Engagement trick:** Every 8 minutes = activity shift or comprehension check (poll/chat/peer)
